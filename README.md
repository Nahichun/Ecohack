# Ecohack
## Экологический хакатон
Кейс 7 Алгоритм-разведчик: карта пластика для капитана 
Описание проблемы: Экспедиция идет на плоту с ограниченной скоростью и маневренностью. Чтобы попасть в ядро мусорного пятна, капитану нужно понимать, где прямо сейчас находятся максимальные скопления пластика. 
Спутниковые данные (Sentinel-2) есть в открытом доступе, но их анализ требует специальных знаний и времени, которых у Федора Конюхова нет. 
Существующие ограничения: 
На борту: ограниченный спутниковый интернет (низкая пропускная способность). 
Оборудование: обычный ноутбук (не кластер). 
Время: экипаж не может тратить часы на обработку снимков. 
Данные: спутниковые снимки могут быть закрыты облачностью. 
Целевые показатели: 
  Автоматическая загрузка снимков Sentinel-2 за последние 3 дня по заданным координатам.
  Расчет индекса плавающего мусора (FDI или аналог).
  Визуализация результатов в виде понятной цветной карты (красный = мусор).
  Время работы скрипта «нажал кнопку — получил результат»: не более 5 минут. 
Задание кейса: Разработать прототип веб-приложения или Jupyter-дашборда, который по введенным координатам района плавания автоматически строит прогностическую карту скоплений пластика на основе спутниковых данных.

Приложение реализует прототип системы оперативного обнаружения зон скопления плавающего пластика по спутниковым данным Sentinel-2.
Пользователь задаёт координаты и радиус анализа.
Система автоматически:
Загружает снимки за последние 3 суток.
Вычисляет индекс FDI (Floating Debris Index).
Маскирует облачность по слою SCL.
Нормализует результат.
Отображает карту концентрации.

В случае недоступности API автоматически активируется демонстрационный режим, который имитирует пространственные скопления мусора. 

Индекс FDI выбран как специализированный спектральный показатель для выявления плавающих объектов в морской среде, так как он снижает влияние отражательной способности воды.
